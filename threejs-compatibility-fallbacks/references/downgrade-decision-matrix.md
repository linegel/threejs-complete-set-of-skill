# Downgrade Decision Matrix

This matrix is usable only after the current user explicitly requests teaching
how to apply fallback when WebGPU is unavailable and the activation record is
complete. It is not a backend router, a mobile optimization guide, or a reason
to modify canonical WebGPU/TSL skills.

Use the first branch that preserves the declared invariant honestly. Every
selection requires a canonical-loss-ledger row, separate artifacts, and
numeric values labelled `Authored`, `Derived`, `Measured`, or `Gated`.

| Unavailable capability or branch constraint | Preferred response | Next response | Last acceptable response | Reject |
| --- | --- | --- | --- | --- |
| WebGPU backend | Precompute canonical outputs when interaction can be removed. | Generate fields, geometry, LUTs, or animation offline/CPU-side with bounded uploads and named lost dynamics. | Quarantined legacy branch after explicit maintenance acceptance. | Any automatic or unrequested branch. |
| Runtime compute | Precompute or stream versioned canonical results. | CPU update only when sustained upload, allocation, and visual-error gates pass. | Remove the dynamic feature. | Presenting static or CPU-limited behavior as canonical GPU simulation. |
| Storage textures or buffers | Use immutable attributes/textures or baked pages with explicit indexing and precision loss. | CPU-generated data with bounded cadence and lifetime. | Remove the stateful feature. | Per-object update/draw expansion hidden as equivalent scalable storage. |
| MRT/shared scene signals | Remove dependent diagnostics/effects and narrow the branch claim. | Serialize explicitly inventoried passes if target timing, tile traffic, and memory gates pass. | Remove the dependent feature. | Private duplicate scene graphs described as one shared pipeline. |
| Scene-linear HDR attachment | Use a lower-precision linear representation with measured radiance error if supported. | Pre-bake the look with an explicit display-referred domain. | Remove HDR-dependent bloom/exposure behavior. | Display-space arithmetic described as scene-linear HDR. |
| Node post graph | Keep one branch-owned output graph and remove optional effects. | Pre-bake a display-referred look with its domain and camera/lighting limits named. | Remove presentation effects. | Duplicate tone maps or output conversions. |
| Built-in node mechanism | Implement a documented branch-specific shader equivalent only when its invariant tests remain valid. | Precompute the contribution. | Remove the feature. | Invented APIs or a visually similar but physically unrelated effect. |
| Dynamic interaction | Bound interaction to authored states or precomputed transitions. | CPU-side approximation with explicit latency and state-domain limits. | Remove interaction. | Canned output described as reactive behavior. |
| Memory pressure | Reduce or remove branch resources with the largest peak-live and traffic contribution, subject to visual-error gates. | Stream or precompute bounded variants with measured upload behavior. | Remove the feature. | Using allocation total alone while ignoring transient peaks and tile traffic. |
| Tile-GPU traffic pressure | Eliminate stored intermediates, resolves, pass breaks, and full-resolution histories in the branch graph. | Reduce attachment/sample/storage pressure under invariant-specific visual gates. | Remove the bandwidth-heavy feature. | Claiming bandwidth relief without a labelled traffic model or counter evidence. |
| Sustained thermal degradation | Reduce branch update or representation cost through an explicit quality state whose visual error passes. | Replace dynamics with static/precomputed data. | Remove the feature. | Accepting a cold run or averaging away final-segment degradation. |
| GPU timestamp unavailable | Rerun on a target/browser that exposes required timing. | Narrow sign-off to claims that do not require GPU attribution. | Return `INSUFFICIENT_EVIDENCE_GPU_TIMING`. | CPU timing, presentation cadence, or a shorter window presented as GPU cost. |
| Weakened invariant exceeds visual gate | Select a higher-fidelity representation. | Narrow the supported camera/lighting/temporal envelope explicitly. | Remove the feature. | Raising the threshold after inspection or hiding error with post treatment. |

## Decision Order

- Confirm explicit activation and canonical reference evidence.
- Preserve the invariant with precomputed canonical output when dynamics are
  unnecessary.
- Move generation offline or CPU-side only with bounded upload, memory, and
  sustained evidence.
- Remove a feature whose invariant cannot survive honestly.
- Build a quarantined legacy renderer branch only after maintenance acceptance.
- Reject any branch that fails visual, sustained performance, resource,
  lifecycle, or evidence-sufficiency gates.

Native WebGPU quality scaling is intentionally absent. When WebGPU is
available, quality selection remains canonical-owner work and this skill is
inactive.
