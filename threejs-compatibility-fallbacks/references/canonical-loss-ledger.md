# Canonical Loss Ledger

Use this ledger only after the current user explicitly requests teaching how to
apply fallback when WebGPU is unavailable. It must never be copied into a
canonical WebGPU/TSL skill. The canonical owner defines truth; this file only records what
an authorized compatibility branch may preserve, weaken, or remove.

Every numeric value uses `{ value, unit, label, source }`, where `label` is
`Authored`, `Derived`, `Measured`, or `Gated`. No row supplies universal
device-class constants. Each branch derives its performance envelope from the
named target refresh after browser and compositor reserves.

## Universal Compatibility Proof Delta

Every authorized branch also records:

- the explicit user request and tested WebGPU-unavailable condition;
- separate canonical and compatibility artifact paths;
- requested presentation rate `Authored`, actual display refresh `Measured`,
  target rate `Gated`, refresh period `Derived`, browser/compositor reserves
  `Measured` or provisional `Authored`, stage envelopes `Derived`, and acceptance
  limits `Gated`;
- cold and sustained CPU/GPU `p50 [Measured]` and `p95 [Measured]`, presentation
  misses, quality transitions, and final stable state;
- resident and peak transient memory, attachment footprint, load/store/resolve
  traffic, upload/readback volume, and bounded traffic model;
- per-invariant visual-error maps, measured distributions, and frozen gates;
- color/data domains, sole tone-map/output owner, disposal, and rebuild rules;
- `INSUFFICIENT_EVIDENCE_GPU_TIMING` whenever GPU attribution is required but
  timestamp evidence is unavailable.

Fallback authorization does not authorize weaker proof.
