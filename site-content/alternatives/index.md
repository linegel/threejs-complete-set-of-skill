## Choose the route before the product

The useful alternatives for Three.js WebGPU work are different kinds of help. Official documentation answers what the library supports. Agent skills package repeatable procedures. MCP tools can expose live state and actions. Courses or human experts can teach or implement. Project rules and prompts carry requirements unique to one codebase.

Choose the category that owns the missing job. Only then compare named options inside that category.

## Option categories

| Need | Strong starting route | Main limitation |
| --- | --- | --- |
| Exact current Three.js API behavior | Official docs and tagged source | Does not implement or validate the user's repository |
| Reusable agent procedure | Agent Skills | Requires a compatible agent and careful skill selection |
| Live scene inspection or action | A reviewed MCP server | Requires runtime integration, permissions, and server-specific trust |
| One isolated request | A precise project prompt | Can become stale or inconsistent when copied as a playbook |
| Organization-specific standards | An internal guide or project rule | The organization owns authoring, review, and maintenance |
| Human curriculum | A course or structured tutorial | May not match the installed revision or production workload |
| Scoped implementation help | A qualified engineer or consultant | Requires a real scope, review process, and cost agreement |

This pack is one route inside the agent-skill category. It should not present itself as an alternative to an MCP server when live tool access is the actual need, or as a replacement for the official API reference.

## Compact decision tree

1. Is the question an exact API, import, or compatibility fact? Start with official Three.js sources.
2. Does an agent need a repeatable implementation and validation procedure? Compare Agent Skills.
3. Does the answer depend on a running scene or external system? Evaluate a narrowly scoped MCP tool.
4. Is the missing knowledge unique to the current project? Improve the project prompt or internal rules.
5. Is the team missing human understanding or execution capacity? Evaluate structured learning or professional help.
6. If two named options remain, use the [comparison hub](/compare/) rather than another alternatives page.

## How options can coexist

An advanced task can use official docs for API truth, a project prompt for local requirements, an Agent Skill for specialist procedure, an MCP tool for authorized live inspection, and a validation harness for proof. The important constraint is ownership. Each source should answer a different question.

Avoid loading several overlapping routers, applying two final-image workflows, or treating a tool's success response as independent validation. Keep one route responsible for architecture and one route responsible for final acceptance.

## Cost and maintenance questions

Do not infer total cost from repository access. A public skill can still require agent usage and engineering time. A free tutorial can still consume substantial learning time. A paid expert can reduce or increase total cost depending on scope and review quality.

Use [pricing](/pricing/) for the pack's exact $0 repository boundary and a cost worksheet. For every external option, follow the current provider or repository source instead of copying volatile prices into this site.

## Limitations

These landscapes are curated, not exhaustive. Inclusion does not certify security or output quality, and omission does not imply poor quality. Mutable facts are bounded by `Last reviewed`; recheck installation, compatibility, maintenance, and license sources before adoption.

If the job has already become a source-to-target change, skip the alternatives stage and use the [migration guides](/migrate/).
